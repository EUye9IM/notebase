# notebase

一个Rust实现的基于Rag技术的笔记库，提供基本的文本/图片管理与自然语言检索。

首先rust现在没有比较好的gguf模型本地运行的库，所以是可以先依赖外部Api，也可以是本地部署的ollama。

然后项目需要是一个服务器-客户端的架构。

因为这个项目预期是一个cli的工具(也可能可以有用户界面)，
在cli的时候如果我插入一个文本，响应时间比较慢然后完成时间比较长是不太能接受的。
比较理想的解决办法是前台接受命令之后退出，然后算向量的流程交给后台程序。

服务端和客户端的消息方式先暂定用命名管道，应该是能双工，但是跨平台需要研究一下。

然后基本的一些命令设计是：
nb add
nb list
nb find
nb show
nb mod
nv delete

nv serve

对短文本直接用嵌入模型，长文本使用聊天模型得到一个摘要在去做嵌入。图片也设计prompt得到摘要描述。

模型选型是嵌入式模型先用qwen3-embedding-4b，聊天模型用qwen3.5-0.8b（思考时间很长要快只有0.8了）

然后数据库先用sqlite实现，先不搞复杂的。

find选择条目的方式是用标准差、和平均值之类的方法选择一些比较关联的项，参数应该可配置。
